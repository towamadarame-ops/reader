# 陪伴阅读（Peiban Reader）

一个温柔的 **AI 陪伴式阅读器**。它把「读小说」和「有人陪你一起读」结合在一起：你在读书，AI 伴侣会在旁边点评、聊天、共享心情，还能为你朗读、总结章节、记录你的想法。

- 🧩 **单文件、纯前端**：核心就是一个 `index.html`，用原生 JS 编写，无需构建、无需后端。
- 📱 **可安装为 App（PWA）**：支持离线打开，添加到手机主屏幕后体验接近原生 App。
- 🔒 **数据本地优先**：所有书籍、聊天记录、密钥、设置都只保存在你自己的浏览器里（localforage / IndexedDB），不上传任何服务器。
- 🤖 **兼容任意 OpenAI 格式接口**：SiliconFlow、DeepSeek、Kimi、Gemini、OpenAI、OpenRouter 等都可直接接入。

---

## 目录结构

| 文件 | 作用 |
| --- | --- |
| `index.html` | 应用本体（界面 + 全部逻辑），直接用浏览器打开即可 |
| `sw.js` | Service Worker，负责离线缓存，让站点满足「可安装 PWA」条件 |
| `minimax-proxy-worker.js` | 可选的 Cloudflare Worker CORS 反代模板，用于语音朗读跨域直连被拦截时 |

## 功能一览

### 📖 阅读器

- **导入本地 TXT 书籍**，支持 `UTF-8` / `GBK` 编码切换
- **书架管理**：可保存多本书，随时切换、删除
- **自动生成目录**：按常见章节标题规则自动解析全书目录
- **两种翻页模式**：上下滚动 / 左右翻页（仿真分栏）
- **阅读进度**：底部进度条 + 当前章节 / 百分比状态栏
- **章节导航面板**：点击屏幕中央呼出，含「上一章 / 下一章」按钮与**可拖动的全书章节滑块**（拖动时实时预览章节名，松手即跳转），并可一键打开完整目录
- **侧边目录**：点选任意章节快速跳转
- **全文搜索**：按关键词检索全书并定位
- **主题定制**：背景色 / 字体色 / 注释高亮色、字号、行距、页边距，还可上传自定义背景图片

### 🤖 AI 阅读陪伴

- **让伴侣点评本章**：AI 结合上下文对当前章节发表看法
- **划词交互菜单**（选中正文后弹出）：
  - 记录想法（把你的批注写在原文旁）
  - 询问伴侣
  - 换你会怎么做（角色代入）
  - 解释这个词
  - 接下来会怎样（剧情猜测）
- **主动陪伴**（可在设置里开关）：
  - 久读关心：连续阅读一段时间后主动提醒你休息
  - 共读心情：进入新章节时分享 TA 的情绪
  - 章节末小互动：读完一章后发起一个小问题
- **章节摘要**：可让 AI 为各章生成摘要（支持批量生成、断点续跑），并作为 AI 的「剧情记忆」
- **书籍笔记**：集中查看你在书中记录的所有想法与 AI 点评，支持导出

### 💬 聊天陪伴

- 与 AI 伴侣自由聊天，聊天内容会结合你当前的阅读进度
- **人设自定义**：伴侣姓名、头像、气泡颜色、人设 Prompt
- **用户设定**：你的姓名、头像、气泡颜色、聊天背景、用户人设
- **拍一拍**：点头像触发互动，后缀文案可自定义
- **角色扮演**：沉浸式扮演（神态 / 动作 / 心理描写）、更主动交流、回复详细度可调
- **世界书（Lorebook）**：可添加带关键词的世界观 / 设定条目，供 AI 参考
- **长期记忆**：可开关，设定 AI 读取最近多少轮对话、每多少轮自动总结一次记忆，记忆库可手动编辑或删除

### 🔊 语音朗读（TTS）

- 接入 MiniMax 语音合成，朗读当前页
- 可设置音色 `voice_id`、模型、语速
- **AI 情绪配音**：由 AI 为文本分段指定情绪，语气更自然生动
- **CORS 反代**：浏览器直连被跨域拦截时，可启用自建 Cloudflare Worker 反代（见 `minimax-proxy-worker.js`）

### ⚙️ 接口与数据

- **聊天 API**：Base URL / API Key / 模型，支持拉取模型列表、测试连接、多套配置预设
- **摘要 API**：可单独配置，或一键「同聊天配置」
- **数据管理**：导出 / 导入全部数据（JSON 备份）、清除聊天记录、恢复默认预设

## 快速开始

### 方式一：本地直接打开
双击 `index.html` 用浏览器打开即可使用。
> 注意：以 `file://` 方式打开时，Service Worker（离线 / PWA 安装）不会生效，但阅读、聊天等功能正常。

### 方式二：部署到静态托管（推荐，可安装为 App）
把这三个文件（`index.html`、`sw.js`、`minimax-proxy-worker.js`）上传到任意静态托管即可，例如 **GitHub Pages**、Cloudflare Pages、Vercel 等。

以 GitHub Pages 为例：
1. 新建仓库，把文件推上去；
2. 仓库 `Settings → Pages`，选择部署分支（如 `main` / 根目录）；
3. 打开生成的网址即可访问，并可在手机浏览器里「添加到主屏幕」安装为 App。

### 配置 AI 接口
1. 打开应用 → 右上角「设置」；
2. 在「聊天 API」填入你的 Base URL、API Key、模型（可点「拉取模型列表 / 测试连接」验证）；
3. 需要朗读功能时，在「语音朗读」里填入 MiniMax 的 Key 等参数；
4. 点击底部「保存设置」。

### （可选）部署语音朗读反代
若语音朗读因浏览器跨域被拦截，可按 `minimax-proxy-worker.js` 文件顶部注释的步骤，在 Cloudflare 免费部署一个 Worker，然后在设置里打开「启用 CORS 反代」并填入 Worker 网址。

## 隐私说明

- 你的 **API Key、Group ID、反代地址、书籍、聊天记录** 等全部只保存在浏览器本地，不会随代码上传，也不会发送到除你所配置的 AI 接口之外的任何服务器。
- 代码本身**不含任何硬编码的密钥或个人信息**，可放心公开部署。
- ⚠️ 提醒：应用内「导出数据」生成的 JSON 备份**会包含你的 API Key**，请勿将该备份文件提交到公开仓库。

## 技术栈

- Service Worker —— 离线缓存与 PWA
- Cloudflare Workers —— 可选的 CORS 反向代理
